# Delnee Travel Data System — Project Summary

## Overview
Delnee is a data-driven system designed to support travel planning by managing structured data related to users, countries, hotels, rooms, and tourist destinations.

The system enables users to explore travel options, filter results based on preferences, and make bookings through a centralized platform.

---

## Objectives
- Simplify travel planning through a unified system  
- Provide structured access to travel-related data  
- Enable dynamic data retrieval using SQL queries  
- Support booking and reservation processes  

---

## System Features
- User authentication (login/logout)
- Country and tourist attraction exploration
- Hotel and room filtering
- Booking system for travel services

---

## Data Perspective
From a data perspective, the system is built around a relational database model that connects multiple entities:

- Countries → Places (tourist attractions)  
- Countries → Hotels → Rooms  
- Users → Bookings  

This structure supports efficient querying and reflects real-world relationships.

---

## Key Learnings
- Designing relational database schemas  
- Writing SQL queries for dynamic data retrieval  
- Structuring data for real-world applications  
- Understanding how frontend inputs connect to backend data systems  

---

## Conclusion
This project demonstrates how structured data and relational databases can be used to build a functional system that supports real-world use cases such as travel planning and booking management.
